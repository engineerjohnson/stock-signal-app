# 收盤後如何看到完整資料（含連次/連量）

## 結論：你需要在盤中把頁面開著

連次/連量是「即時累積」的指標，無法事後從任何免費 API 還原。
只要盤中有開著頁面，收盤後的所有 tab 資料都會是完整的。

---

## 為什麼連次/連量無法事後取得？

TWSE（台灣證交所）的公開 API 只提供每日彙總資料：

| 能取得 | 無法取得 |
|--------|---------|
| 收盤價 | 每一筆成交的方向（漲/跌）|
| 開盤價 | 連續漲了幾次 |
| 成交量（總） | 連漲期間累積了多少張 |
| 漲跌幅 | 盤中任何時間點的價格序列 |

連次/連量需要「從頭到尾觀察每一筆成交的方向變化」，
就像比賽結束後你只知道最終比分，無法推算最後連續得了幾分。

---

## 正確的使用方式

### 盤中（09:00 ~ 13:30）

1. **開盤前開啟頁面**（08:50 左右即可）
2. 頁面保持開著，系統每 3 秒自動抓一次即時報價
3. 連次/連量會從第一筆成交開始自動累積
4. 狀態列顯示「⏳ 累積資料中…」代表前幾次掃描還在建立基礎，屬正常

### 收盤後（13:30 之後）

1. 資料會自動儲存在瀏覽器的 **localStorage**
2. 關掉頁面後重新開啟，資料依然保留
3. 所有 tab 都能正常篩選（強勢、弱勢、飆風、主投買進…等）
4. 收盤 banner 會顯示「資料 HH:MM」，讓你確認資料的時間點

## 各情況說明

| 情況 | all tab | 其他 tab（強勢/飆風等）|
|------|---------|----------------------|
| 盤中頁面開著 ✅ | 有資料 | 有資料，即時更新 |
| 盤中開著，收盤後重開頁面 ✅ | 有資料 | 有資料（今日收盤快照）|
| 今天第一次開，已收盤 ❌ | 有資料（昨日）| 連次/連量 = 0，大多為空 |
| 昨天開過，今天盤後開 ⚠️ | 有資料 | 有資料（昨日快照，標示「前次收盤資料」）|

---

## 實際操作建議

```
每個交易日：

08:50  打開頁面
09:00  開盤，系統開始累積
13:30  收盤，資料自動儲存
       → 之後隨時開啟都能看完整的今日訊號
```

### 如果你無法全天開著頁面

- 至少開盤後開著幾個小時，讓主要訊號累積起來
- 12:00 之後的資料較完整（量比時間修正因子接近 1，數值最準）
- 收盤前 13:00～13:30 的快照是最完整的

## 每個 tab 需要的資料來源

| Tab | 需要 | 說明 |
|-----|------|------|
| 全部 | 昨日基礎資料 | 隨時都有 |
| 強勢 / 弱勢 | 連次 ≥ 門檻 + 今日漲跌 | 需盤中累積 |
| 量增價漲 | 量比 ≥ 2x + 漲幅 ≥ 1% | 需盤中掃描 |
| 飆風向上 | 連次 + 連量 + 量比 | 需盤中累積 |
| 主投買進 | 連次 + 連量 ≥ 200 張 | 需盤中累積 |
| 近漲停 / 近跌停 | 漲跌幅 ≥ 9% | 需盤中掃描 |
| 推薦飆股 | 連次 + 連量 + 量比 | 需盤中累積 |
| 自選股 | 自選清單 + 基礎資料 | 隨時都有 |

## 常見問題

**Q：為什麼 all tab 有資料，其他 tab 都空的？**
A：今天沒有在盤中開啟過頁面，沒有連次/連量資料可供篩選。
明天開盤時把頁面打開，收盤後就會有完整資料。

**Q：localStorage 什麼時候會清空？**
A：手動清除瀏覽器資料，或換瀏覽器/裝置時。同一台電腦的同一瀏覽器，資料會一直保留到被覆蓋為止（每次掃描都會更新）。

**Q：可以在手機上看嗎？**
A：可以，但手機瀏覽器背景時會暫停 JavaScript，等於頁面沒有開著，連次無法累積。建議在電腦上使用。
